## üîë Key Concepts

### Tags and Elements
```html
<tagname>Content</tagname>
```
- Opening tag: `<p>`
- Closing tag: `</p>`
- Self-closing: `<img />` or `<br />`

### Attributes
```html
<a href="https://google.com" target="_blank">Link</a>
```
- `href` and `target` are attributes
- Provide extra information about elements

### Semantic HTML
Use tags that describe their content:
- ‚ùå Bad: `<div>Header</div>`
- ‚úÖ Good: `<header>Header content</header>`

**Why?** Accessibility, SEO, and code readability.

## üé® Don't Worry About Appearance

**Your HTML will look ugly!** That's completely normal and expected.

HTML = **Structure**
CSS = **Style** (you'll learn this next week)

**For now:**
- Focus on getting the structure right
- Use semantic tags appropriately
- Don't worry about colors, fonts, spacing
- Ugly but well-structured > pretty but broken

---

## ‚úÖ How to Know You're Ready for CSS

You should be comfortable:
- Writing HTML from scratch (not copy-paste)
- Using semantic tags appropriately
- Creating forms and tables
- Understanding attributes (href, src, alt, etc.)
- Reading HTML and understanding its structure

**You don't need to memorize every tag!** Just understand the common ones and know how to look up others.
